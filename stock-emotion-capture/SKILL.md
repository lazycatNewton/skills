# A股情绪复盘

## 核心定位

将本技能视为 A 股情绪复盘工作流，而不是交易信号生成器。输出应是可解释的每日复盘和精简的次日观察池。不要提供直接买入或卖出指令。

## 数据流程

1. 明确交易日期，格式为 `YYYYMMDD`。如果用户使用“今天”等相对日期，必须同时写出具体日期。
2. 以 `mcp_router.get_limit_pool` 且 `pool="both"` 作为涨停/跌停数据的主数据源。
3. 获取涨跌停数据后，必须先整理行业/板块视图和连板梯队视图，再进入个股筛选。
4. 需要指数背景时，使用 `mcp_router.get_market_index`。
5. 只有在需要历史确认时，才使用 `mcp_router.get_historical_data`。
6. 将涨跌停池中的 `industry` 作为第一层行业/题材字段。如果没有更强的概念数据源，必须明确说明。
7. 不要编造缺失字段、题材、板块地位或上涨原因。不确定判断必须标注为推断。

字段定义和派生指标见 `references/mcp_router_contract.md`。

## 复盘流程

按以下顺序分析：

1. 市场情绪：涨停数、跌停数、连板高度、连板数量、炸板风险、封板质量、指数背景。
2. 板块情绪：行业/题材集中度、连板股分布、龙头/中军/跟风结构、持续性、分歧或退潮信号。
3. 个股强度：封板时间、回封质量、封单强度、换手率、成交额、连板高度、ST/次新风险。
4. 风险过滤：剔除或降级 ST、异常次新、尾盘弱封、反复炸板、封单明显不足、孤立无板块支撑的个股。
5. 次日预案：输出次日观察条件和失效条件，而不是买入建议。

评分口径和报告格式见 `references/review_framework.md`。

## 必需输出

除非用户另有要求，否则按以下顺序输出：

1. 市场情绪结论
2. 主线板块复盘
3. 连板梯队
4. 涨停结构分析
5. 风险信号
6. 次日观察池
7. 个股入选理由与失效条件

观察池要保持精简。默认保留 5-15 只候选，并按 `核心观察`、`重点观察`、`风险观察`、`剔除` 分组。

## 完整性规则

- 避免未来函数：不得用次日结果解释当日入选逻辑。
- 保持原始数据和加工结论的边界：实际 MCP 字段、派生指标、主观推断必须分开表达。
- 明确说明数据缺口，尤其是板块概念、封单口径、实时/收盘口径、节假日或未收盘数据。
- 优先使用简单、可审计的规则，再考虑复杂评分。
- 所有结论都必须能追溯到源数据字段，或追溯到明确命名的派生计算。
