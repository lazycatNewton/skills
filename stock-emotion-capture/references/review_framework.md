# 复盘框架

使用本参考文档，将涨跌停池数据整理为结构化情绪复盘。

## 数据预处理

获取 `mcp_router.get_limit_pool` 指定日期的涨停/跌停数据后，先生成两个基础视图，再做个股判断：

- 行业/板块视图：按 `industry` 汇总涨停数量、连板数量、最高连板高度、跌停数量，用于判断情绪集中度和亏钱效应分布。
- 连板梯队视图：按 `consecutive_limit_up_days` 汇总涨停股，标出最高板、二板、首板，以及各梯队对应行业。

不要在完成这两个基础视图前直接给出观察池。

## 市场情绪

评估以下内容：

- 涨停数量和跌停数量。
- 连板高度：取最高的 `consecutive_limit_up_days`。
- 连板梯队：按 `consecutive_limit_up_days` 对涨停股分组。
- 封板质量：更早的 `first_limit_time`、更稳定的 `last_limit_time`、更低的 `break_limit_count`、更强的 `seal_strength`。
- 亏钱效应：跌停数量、连续跌停、反复开板行为。
- 指数背景：如有指数数据，可作为背景使用，但要与涨停池结论分开。

## 板块情绪

先从 `industry` 字段入手：

- 按行业统计涨停股数量。
- 按行业统计连板股数量。
- 判断当日情绪是集中、轮动，还是分散。
- 用连板高度、封板时间、封单强度、成交额区分潜在龙头和跟风。
- 如果只有行业数据，写作“行业口径”，不要声称已经确认精确概念主线。

## 个股强度

按以下维度进行定性评分：

- 连板强度：`consecutive_limit_up_days` 越高，强度越高，但风险也更高。
- 封板效率：更早的 `first_limit_time`，以及更少的 `break_limit_count`。
- 回封质量：首次封板和最后封板时间差过大，或炸板次数过多，会降低可信度。
- 封单强度：比较 `seal_fund` 与 `free_float_market_cap`。
- 换手健康度：极端缩量或极端高换手都需要谨慎。
- 板块地位：结合行业集中度和连板梯队位置判断。
- 风险惩罚：ST、次新股、尾盘弱封、反复炸板、孤立行业、高波动异常等。

## 观察池

使用以下分组：

- `核心观察`: 板块支撑强、封板质量干净、梯队地位清晰。
- `重点观察`: 强度较好，但仍有一两个不确定因素。
- `风险观察`: 有关注价值，但存在高波动、弱封、反复炸板、次新/ST/特殊风险，或市场环境较差。
- `剔除`: 源数据风险明显，或结构强度不足，不适合纳入次日观察。

每只入选股票必须包含：

- 股票代码和名称。
- 行业口径。
- 入选理由。
- 主要风险。
- 次日观察条件。
- 失效条件。

## 默认报告结构

```text
1. 市场情绪结论
2. 主线板块复盘
3. 连板梯队
4. 涨停结构分析
5. 风险信号
6. 次日观察池
7. 个股入选理由与失效条件
```

## 严格禁止

- 不要把输出称为“买入推荐”。
- 不要隐藏板块/题材数据不完整造成的不确定性。
- 不要用次日价格表现解释当日入选逻辑。
- 不要把实际字段和派生计算混成一个无法区分的结论。
