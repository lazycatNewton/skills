# Chan Theory and Implementation Differences

归档时间：2026-07-01

## 结论

本项目当前接受 `chan/references/notion.md` 中严格缠论定义与 `chan/scripts/chan_core.py` 中日线笔级别简化实现之间的差异。当前实现目标不是完整复刻多级别递归缠论，而是在 A 股前复权日 K 数据上提供可解释、可视化、可复现的第一阶段分析能力。

## 背驰定义差异

`notion.md` 中的趋势背驰要求完整结构：

```text
a + A + b + B + c
```

其中 A、B 为同级别中枢，b、c 为连接中枢的同向次级别走势段。严格背驰还要求 c 段力度小于 b 段力度，并要求 MACD DIF/DEA 回拉至零轴附近。

当前代码没有完整匹配 `a+A+b+B+c`，也没有确认中枢 A、B 是否属于同一趋势中的同级别连续中枢。代码使用 `center_count >= 2` 作为结构近似，并结合同类分型 DIF 背离或同向笔 MACD 柱面积衰竭标记 `top_exhaustion` / `bottom_exhaustion`。

## 背离与背驰边界差异

`notion.md` 明确区分背离与背驰：背离是价格与指标不确认，背驰是趋势力度衰竭结论。单纯价格创新高或新低而 MACD 未确认，只能视为背离信号。

当前代码在全局中枢数量不少于 2 时，会将同类分型 DIF 背离升级为 `BC-S` / `BC-B`。因此当前图表中的 `BC-S` / `BC-B` 应理解为“背驰近似标记”或“力度衰竭观察标记”，不是完整理论背驰。

## MACD 零轴条件差异

`notion.md` 要求 MACD 黄白线回拉至零轴附近，以体现中枢引力场作用。

当前代码计算 DIF、DEA 和 MACD hist，并使用 hist 面积比较走势力度，但没有量化或验证 DIF/DEA 是否回拉零轴附近。

## 第一类买卖点差异

`notion.md` 定义一买/一卖为下跌或上涨趋势中最后一个中枢之后形成的底背驰点或顶背驰点。

当前代码只在一笔结束点存在 `bottom_exhaustion` / `top_exhaustion` 时标记 `1B` / `1S`。代码没有确认该点是否位于最后一个中枢之后，也没有确认完整趋势结构或多级别递归关系。

## 第二类买卖点差异

`notion.md` 定义二买/二卖为一买/一卖后首次次级别回抽或反弹不创新低或新高，并指出二买/二卖由次级别一买/一卖构成。

当前代码用后续笔近似次级别走势：一买后寻找后续第一根满足 `stroke.low > first_buy.price` 的下跌笔作为 `2B`，一卖后寻找后续第一根满足 `stroke.high < first_sell.price` 的上涨笔作为 `2S`。代码没有验证次级别一买/一卖，也没有严格确认“首次有效回抽/反弹”。

## 第三类买卖点差异

`notion.md` 定义三买/三卖为离开中枢后回抽或反弹不回到中枢区间：三买要求低点大于 ZG，三卖要求高点小于 ZD。

当前代码与该定义较接近，但仍以“一笔”近似“次级别走势”。代码没有确认离开中枢和回抽/反弹是否构成完整次级别走势，也没有做多级别结构确认。

## 盘整背驰差异

`notion.md` 区分趋势背驰与盘整背驰。

当前代码没有单独实现盘整背驰类型，也没有输出 `consolidation_exhaustion` 一类信号。普通 `DIV-B` / `DIV-S` 仅作为观察提示，不等价于盘整背驰。

## 当前接受的实现边界

- 主级别使用日 K。
- 不做包含处理，每根原始 K 线直接进入图表和分析层。
- 一笔规则为至少 3 个基本 K 线单位。
- `DIV-B` / `DIV-S` 表示普通背离观察信号。
- `BC-B` / `BC-S` 表示当前代码中的背驰近似或力度衰竭标记。
- `1B` / `1S` 只由一笔结束点上的 `bottom_exhaustion` / `top_exhaustion` 触发。
- `2B` / `2S`、`3B` / `3S` 使用笔级别近似，不做多级别递归确认。
